Can we implement an algorithm to compute $\sqrt{2}$?\\
\vspace{0.05in}

\textcolor{gray}{\bf\large Algorithm}\\
\vspace{0.05in}
Babylonian method:
\begin{align*}
 &x_0 \approx \sqrt{S} && \text{\# Start with a initial guess}\\
&x_{n+1} =\frac{1}{2}\left(x_n+\frac{S}{x_n}\right)& & \text{\# iteration or updating function}\\
& \lim_{n\rightarrow\infty} x_{n}=\sqrt{S} &&    \text{\# theory}
\end{align*}

The basic idea:
\begin{itemize}
\item if $x_n>\sqrt{S}$ (overestimate), then $\displaystyle \frac{{S}}{x}< \sqrt{S}$ (underestimate) 
\item if $x_n<\sqrt{S}$  (underestimate), then $\displaystyle \frac{{S}}{x}< \sqrt{S}$ (overestimate)
\item anyway, the average $ \displaystyle x_{n+1} =\frac{1}{2}\left(x_n+\frac{S}{x_n}\right)$ is a better estimate
\end{itemize}

{\bf Remark}: we will not let the iteration go to $\infty$, instead, we'll stop at $N$. 

{\bf Stop Criteria}: 
we stop the iteration when
$$
|x_{n+1}-x_{n}| < \text{tol},~~(\text{tol is a small number for tollerance})
$$
